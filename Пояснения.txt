В начале необходимо выбрать исполняемый файл тестируемой программы.

Добавление аргументов (скорее всего переименую в "параметры"):
Можно добавить в конец, а также после или перед текущим (просматриваемым сейчас), когда есть хотя бы один.
Выбор параметров для редактирования можно производить с помощью списка или кнопок перемещения "вверх"/"вниз".

Название аргумента никак не используется программой, оно было добавлено для возможности лёгкой идентификации разных параметров в списке.
Тип аргумента:
Входные аргументы вводятся в консоль тестируемой программы, каждый на отдельной строке.
Выходные аргументы - это ожидаемый вывод программы.

Тип содержания - на основе чего будет генерироваться входной параметр/чему должен соответствовать вывод программы.
Пустой - пустая строка (добавлен временно для своего удобства)
Текст - точное соответствие тексту (нет случайной генерации входных данных; для выходных - вывод программы должен совпадать с текстом).
Регулярное выражение - вывод должен удовлетворять рег.выр. Для входного параметра будет генерироваться случайная строка, удовлетворяющая рег.выр. (пока не реализовано, была пара проблем).

В будущем могу добавить ещё пару вариантов (как числа в заданном диапазоне).

Ниже кнопки удаления параметра и его сдвига вверх/вниз в списке.
Все аргументы "исполняются" в указанном порядке (то есть можно настроить, чтобы система ожидала вывод тестируемой программой каких-то промежуточных результатов).

Планирую также показывать сбоку весь список и указатель на текущий аргумент, а то при сдвиге можно запутаться.

Использование предыдущих ошибок пока не реализовано. Для каждой тестируемой программы отдельно будут сохраняться все тесты (конкретные значения), которые она провалила, чтобы их можно было легко использовать после внесения исправлений в неё, чтобы убедиться, что теперь они проходят успешно.

Количество тестов - сколько успешных тестов должна пройти программа. Пока бесполезно, потому что ещё не реализованы случайно генерируемые входные данные (но сама по себе функция работает).


После проверки выводится либо сообщение об успехе, либо сообщение о неудаче (а также все входные данные и вывод программы - скорее всего оформлю получше), либо сообщение об ошибке, если, например, исполняемый файл не удалось запустить.


В папке 'example' есть пара программ, на которых можно провести тесты:
gcd вычисляет НОД двух чисел, введённых в одной строке; результат выводит на следующей.
echo-3 просит ввести строку и сразу её выводит (повторяется три раза).
Обе намеренно написаны небрежно и без проверок, чтобы можно было посмотреть на проваленые тесты (echo-3, например, плохо реагирует на пробелы в вводимых строках).
